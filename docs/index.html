~/sublime-tab-organizer

A tiny CLI + companion plugin that lets you list, merge, sort, dedupe, and persist Sublime Text tabs from the terminal. Stop dragging tabs between windows — script it.

$ git clone https://github.com/GindaChen/sublime-tab-organizer && cd sublime-tab-organizer && ./install.sh

features

Each command is one subcommand of sto. Run sto <cmd> --help for per-command examples.

sto list

Every window and every tab with view IDs. ● marks unsaved buffers.

* window 3  (12 tabs)
    ● 14522  src/main.py
      14523  [untitled]

sto merge

Pulls tabs from every window into one. --copy-unsaved migrates dirty/untitled buffers too.

sto merge --copy-unsaved
merged 27 tabs into window 3
copied (untitled) [scratch]

sto dump-untitled

Persist untitled buffers to real files. Timestamp + slug naming. Content-hash dedup means reruns don't pile up copies.

~/sublime-scratch/
  20260423_091204_todo-refactor-auth.py
  20260423_091204_meeting-notes.md

sto sort

Re-order tabs in the active group by name, path, or ext.

sto sort --by path

sto dedupe

If the same file is open in multiple tabs, keeps the first and closes the rest.

sto dedupe
removed duplicate src/util.py
1 duplicates removed

sto close

Close by view id, glob pattern, or "every clean tab."

sto close --pattern '*.log'
sto close --saved

install

Clone, run one script, restart Sublime. macOS paths are the default; Linux/Windows set STO_PLUGIN_DIR.

$ git clone https://github.com/GindaChen/sublime-tab-organizer.git cd sublime-tab-organizer ./install.sh

After install: restart Sublime Text once so the plugin loads, then run sto ping — expect {"pong": true, "version": "0.1.0"}.

architecture

ST's plugin API only runs inside the ST process. The CLI talks to a companion plugin over a local TCP socket; the plugin dispatches commands on ST's main thread.

  your shell                      Sublime Text
  ┌──────────┐   TCP 9998    ┌──────────────────────┐
  │   sto    │ ────────────→ │  SublimeTabOrganizer │
  │   CLI    │ ←──────────── │   (plugin, python)   │
  └──────────┘    JSON/L     └──────────────────────┘

No cross-window move API. Confirmed against the official Sublime API reference: set_view_index, set_sheet_index, move_sheets_to_group are all group-scoped within a single window. merge re-opens files by path for saved tabs; --copy-unsaved re-materializes the buffer in a new view (text + syntax survive; undo history does not).

caveats

  • macOS paths assumed in install.sh. Override with STO_PLUGIN_DIR on Linux/Windows.
  • sto close on a dirty tab triggers ST's native save-prompt dialog. Use --saved to skip.
  • TCP server binds to 127.0.0.1 only — no auth. Any local process can reach it. Fine for personal use; avoid on shared machines.
  • --copy-unsaved loses undo history and view-local state (folds, bookmarks). Text and syntax are preserved.
