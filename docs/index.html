~/sublime-tab-organizer

Stop dragging tabs between Sublime Text windows. Clean your Sublime using cli.

$ git clone https://github.com/GindaChen/sublime-tab-organizer && cd sublime-tab-organizer && ./install.sh
see the commands github

commands

Each subcommand of sto. Shown with a real invocation and output. Run sto <cmd> --help for full flags.

sto list [--json]

Every window and every tab with view IDs. * marks the active window, ● marks unsaved buffers. Pipe --json into jq for scripting.

$ sto list
* window 3  (8 tabs)
        14521  /Users/you/proj/src/main.py
      ● 14522  /Users/you/proj/tests/test_main.py
        14523  [untitled]
  window 5  (4 tabs)
        14600  /Users/you/notes/meeting.md
sto merge [--into N] [--copy-unsaved]

Pulls saved tabs from every window into the active (or --into) window, then closes the emptied source windows. --copy-unsaved migrates dirty/untitled buffers too.

$ sto merge --copy-unsaved
merged 27 tabs into window 3
copied (untitled) [scratch]
copied (dirty) /Users/you/proj/TODO.md
sto dump-untitled [--dir] [--close-source] [--open-saved] [--include-dirty]

Persist untitled buffers to disk with YYYYMMDD_HHMMSS_<slug>.<ext> naming. Slug = first non-blank line. Ext inferred from syntax. Content-sha256 dedup means reruns don't pile up copies.

$ sto dump-untitled --close-source --open-saved
saved   ~/sublime-scratch/20260423_091204_todo-refactor.md
saved   ~/sublime-scratch/20260423_091204_meeting-notes.md
reused  ~/sublime-scratch/20260422_145312_untitled.txt
2 new, 1 reused (content-dedup)
sto close [ids...] [--pattern GLOB] [--saved]

Close tabs by view id, fnmatch glob against the file path, or every clean tab. Dirty tabs trigger ST's native save dialog; use --saved to skip them.

$ sto close --pattern '*/node_modules/*'
closed /Users/you/proj/node_modules/foo.js
closed /Users/you/proj/node_modules/bar.js
2 tabs closed

$ sto close --saved
18 tabs closed
sto sort [--by name|path|ext]

Re-order tabs in the active group. path groups by directory; ext groups by extension, then path.

$ sto sort --by path
sorted 12 tabs by path
sto dedupe

If the same file path is open in multiple tabs, keeps the first seen and closes the rest.

$ sto dedupe
removed duplicate /Users/you/proj/src/util.py
1 duplicates removed
sto group-by-folder

Splits tabs across new windows by project folder. Keeps tabs under the first folder in the active window; moves tabs under each other folder into their own new window.

$ sto group-by-folder
/Users/you/proj-b -> 5 tabs
/Users/you/proj-c -> 3 tabs
sto ping

Health check. Exits non-zero if the plugin isn't reachable.

$ sto ping
{"pong": true, "version": "0.1.0", "windows": 2}

install

One script. Restart Sublime once. Then sto ping.

$ git clone https://github.com/GindaChen/sublime-tab-organizer.git cd sublime-tab-organizer ./install.sh

After install: restart Sublime Text so the plugin loads. Expect {"pong": true, "version": "0.1.0"} from sto ping. macOS paths are the default; Linux/Windows set STO_PLUGIN_DIR.

architecture

ST's plugin API only runs inside the ST process. The CLI talks to a companion plugin over a local TCP socket; the plugin dispatches commands on ST's main thread.

  your shell                      Sublime Text
  ┌──────────┐   TCP 9998    ┌──────────────────────┐
  │   sto    │ ────────────→ │  SublimeTabOrganizer │
  │   CLI    │ ←──────────── │   (plugin, python)   │
  └──────────┘    JSON/L     └──────────────────────┘

No cross-window move API. Confirmed against the official Sublime API reference: set_view_index, set_sheet_index, move_sheets_to_group are all group-scoped within a single window. merge re-opens files by path for saved tabs; --copy-unsaved re-materializes the buffer in a new view (text + syntax survive; undo history does not).

caveats

  • macOS paths assumed in install.sh. Override with STO_PLUGIN_DIR on Linux/Windows.
  • sto close on a dirty tab triggers ST's native save-prompt dialog. Use --saved to skip.
  • TCP server binds to 127.0.0.1 only — no auth. Any local process can reach it. Fine for personal use; avoid on shared machines.
  • --copy-unsaved loses undo history and view-local state (folds, bookmarks). Text and syntax are preserved.
